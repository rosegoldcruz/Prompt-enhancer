Usage Guidelines & Best Practices

✓ Do

  • • Start with clear intent - even simple prompts like "add tests" work well
  • • Be specific about scope - mention specific files or components when relevant
  • • Use domain language - technical terms help the enhancer understand context
  • • Review enhanced prompts - edit further if needed
  • • Iterate if needed - enhance the same prompt multiple times for different approaches

✗ Don't

  • • Write overly vague prompts like "fix stuff" or "make it better"
  • • Ignore context settings - they significantly improve enhancement quality
  • • Expect perfect results every time - review and refine as needed
  • • Use for sensitive or confidential information
  • • Forget to test enhanced prompts in your actual workflow

Troubleshooting Common Issues

Ctrl+P doesn't work

Ensure you have text in the input box and you're in Normal mode (not in a menu or other mode).

Check that you're using the correct key combination for your terminal/browser.

Enhancement takes too long

Press Esc to cancel and try again with a shorter or simpler prompt.

Check your network connection if using external services.

Enhanced prompt isn't helpful

Edit the enhanced prompt manually - it provides a starting point, not a final answer.

Try starting with a more specific initial prompt or adjust your context settings.

Context not detected correctly

Manually select your project type and framework from the context panel.

Add team conventions for more accurate context-aware enhancements.
